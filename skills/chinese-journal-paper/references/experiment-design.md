# 实验设计模板与常见评估指标

## 实验设计框架

### 基本要素

一个完整的实验部分需要回答以下问题：
1. 在什么数据上验证？（数据集）
2. 和谁比？（基线方法）
3. 怎么衡量？（评价指标）
4. 什么条件下做的？（实验设置）
5. 结果如何？（主实验）
6. 为什么有效？（消融实验 + 分析）

### 数据集选择建议

| 任务类型 | 常用数据集 |
|---------|-----------|
| 文本分类 | THUCNews, Fudan Corpus, SST-2, AGNews |
| 命名实体识别 | MSRA NER, People's Daily, OntoNotes |
| 关系抽取 | SanWen, DuIE, TACRED |
| 情感分析 | ChnSentiCorp, SEMEval, NLPCC |
| 机器翻译 | WMT, NIST中英 |
| 问答系统 | CMRc2018, SQuAD, DuReader |
| 知识图谱 | FB15K-237, WN18RR, CN-DBPedia |
| 摘要生成 | LCSTS, CNN/DailyMail |
| 对话系统 | DSTC, Ubuntu Dialogue |
| 图像分类 | ImageNet, CIFAR-10/100 |
| 目标检测 | COCO, VOC |
| 推荐系统 | MovieLens, Amazon Review |

### 基线方法选择原则

- **经典方法**：该任务的基础方法（如 TF-IDF + SVM 用于文本分类）
- **主流方法**：当前该领域的代表性方法
- **最新方法**：近1-2年的SOTA方法
- **消融对比**：本文方法的变体（去掉某模块）

基线数量：
- 核心期刊：3-5 个基线
- 普刊：1-3 个基线

### 实验设置描述模板

```
本文实验使用 PyTorch/TensorFlow 框架，在 NVIDIA GPU（型号）上进行。
优化器采用 Adam/AdamW，初始学习率为 Xe-X，batch size 为 X。
训练轮次为 X epochs，采用早停策略（patience=X）。
词向量使用预训练的 XX 模型（维度为 X）。
对于每个实验，报告 X 次独立运行的平均值和标准差。
```

## 常见评估指标

### 分类任务

| 指标 | 公式 | 适用场景 |
|------|------|---------|
| 准确率（Accuracy） | 正确数/总数 | 均衡数据集 |
| 精确率（Precision） | TP/(TP+FP) | 关注误报 |
| 召回率（Recall） | TP/(TP+FN) | 关注漏报 |
| F1值 | 2×P×R/(P+R) | 综合评价 |
| Macro-F1 | 各类F1的平均 | 多分类均衡 |
| Micro-F1 | 全局TP/FP/FN计算 | 多分类不均衡 |

### 回归任务

| 指标 | 适用场景 |
|------|---------|
| MSE（均方误差） | 通用回归 |
| RMSE（均方根误差） | 与原始量纲一致 |
| MAE（平均绝对误差） | 对异常值不敏感 |
| R²（决定系数） | 模型解释力 |

### 生成任务

| 指标 | 适用场景 |
|------|---------|
| BLEU | 机器翻译、文本生成 |
| ROUGE-L | 文本摘要 |
| METEOR | 机器翻译（考虑同义词） |
| BERTScore | 语义相似度 |

### 排序/检索任务

| 指标 | 适用场景 |
|------|---------|
| MRR | 平均倒数排名 |
| MAP | 平均精度均值 |
| NDCG@K | 归一化折损累积增益 |
| Hit@K | 命中率 |

### 推荐系统

| 指标 | 适用场景 |
|------|---------|
| Recall@K | Top-K召回率 |
| NDCG@K | 排序质量 |
| AUC | 二分类排序 |
| HR@K | 命中率 |

## 消融实验设计

消融实验验证每个模块的贡献，设计原则：

1. **Full Model**：完整模型
2. **w/o Module A**：去掉模块A
3. **w/o Module B**：去掉模块B
4. **w/o Module A+B**：同时去掉A和B（如有关联）
5. **Replace Module A**：用简单替代方案替换A

每个消融对应一条贡献声明：
- "去掉多头注意力后F1下降了3.2%，验证了注意力机制的有效性"
- "用均值池化替换注意力池化后效果下降，说明注意力加权更优"

## 统计显著性

核心期刊建议报告：
- 多次运行的均值 ± 标准差
- 显著性检验（t-test，p < 0.05）
- 效果提升的绝对值和相对百分比

## 实验结果分析模板

```
从表X可以看出：
（1）本文方法在XX指标上达到了XX%，相比最优基线方法XX提升了X个百分点。
（2）相比传统的XX方法，提升尤为明显（X% → X%），主要原因是……。
（3）消融实验（表Y）表明，模块A贡献了最大的性能提升（-X%），验证了……的有效性。
（4）在XX子任务/数据集上，本文方法的表现略低于XX方法，这可能是因为……。
```

## 无实验数据时的处理

如果没有真实实验数据：
1. 用"[待补充]"明确标注
2. 给出建议的实验设计方案
3. 列出推荐的数据集和基线方法
4. 预估可能的实验结果范围（基于方法原理的合理推断）
5. 不要编造实验数据——这是学术不端
