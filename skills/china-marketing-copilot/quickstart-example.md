# 快速上手指南

## 这个Skill能做什么

一句话：帮你在中国3C市场做营销策划，从创意到风控。

具体来说：

| 你说 | 我做 | 输出 |
|------|------|------|
| "帮我们的降噪耳机想个小红书创意" | 加载耳机知识→生成3个创意→风险评估→评论区模拟 | 创意方案 |
| "小米17 Ultra和vivo X300 Pro影像谁强" | 加载手机知识→对比参数和评测数据→给出结论 | 洞察报告 |
| "我们笔记本新品这个传播方案会不会翻车" | 加载笔记本知识→按维度检查风险→模拟负面评论 | 风险评估 |
| "处理这个评测字幕文件" | 纠错→提取关键发现→更新知识库 | 知识库更新 |
| "我们的AR眼镜怎么破局" | 诊断品类认知→调用破局方法论→设计传播方案 | 破局方案 |

## 差异化：为什么不是随便一个AI就行

| 维度 | 通用AI | 本Skill |
|------|--------|---------|
| 数据 | 可能编造 | 只用知识库数据，没有就说没有 |
| 风控 | "注意风险" | 模拟5类评论区人群的负面反应 |
| 行业认知 | 不知道"恰饭""火龙"是什么 | 自带行业黑话/梗/翻车案例 |
| 话术 | AI腔/通稿腔 | 去AI化规则，输出可直用 |
| KOL | 不了解中国数码圈 | 知道谁权威、谁恰饭、谁会打脸 |

## 使用示例

### 示例1：创意策划

**输入**：
```
帮我们降噪耳机打个小红书种草，预算有限
```

**处理流程**：
1. 识别任务类型：CREATIVE
2. 加载 headphones/_index.md（耳机品牌矩阵+横评数据）
3. 加载 ecosystem/industry-memes.md（避雷烂梗）
4. 加载 references/comment-personas.md（评论区模拟）
5. 生成创意 → 风险评估 → 输出

**输出片段**：
```
创意 1: 地铁里的安静实验

核心idea: 降噪一开，地铁变图书馆
传播形式: 小红书图文
目标受众: 25-35岁通勤白领

创意详情:
拍一组对比图——同一节地铁车厢，不戴耳机 vs 戴降噪耳机。
用分贝仪实测地铁噪音（实测数据来自评测KOL XX：95dB→28dB），
配文"你以为地铁很吵？戴上它，地铁变成图书馆"。

数据支撑:
类似案例: 索尼WH-1000XM4地铁降噪对比（B站 XX UP主）
技术事实核查: ✅ 降噪深度28dB来自实测数据

风险评估:
技术事实: ⚠️ 降噪数据来自单KOL，需交叉验证
竞品反击: ✅ 主竞品降噪深度相近，无明显反击点

预期评论区:
- 参数驱动型: "28dB？XX牌子能到35dB" ⚠️
- 普通用户型: "通勤党需要！多少钱？"
```

### 示例2：风险评估

**输入**：
```
我们准备发一条微博说"XX手机续航屠榜"，会不会翻车
```

**处理流程**：
1. 识别任务类型：RISK
2. 加载 mobile/_index.md（检查传播风险提示）
3. 加载 ecosystem/industry-memes.md（"屠榜"是已烂梗）
4. 模拟评论区反应

**输出片段**：
```
风险评估报告

检查维度:
| 维度 | 判定 | 说明 |
|------|------|------|
| 行业黑话 | 🔴 | "屠榜"是已烂梗，用户审美疲劳 |
| 技术事实 | ⚠️ | 需确认"续航第一"的数据来源 |
| 竞品反击 | ⚠️ | 竞品可能在其他场景续航更好 |

评论区风险模拟:
1. "又屠榜了，上次屠榜的是XX" — 来自: 解构找茬型 — 概率: 高
2. "实测续航和宣传差了2小时" — 来自: 参数驱动型 — 概率: 中

总体判定: 高
建议动作: 调整后执行——"屠榜"换成具体数据（如"续航XX小时"）
```

## 数据导入

本Skill的知识越丰富，输出越精准。推荐数据来源：

| 数据类型 | 推荐来源 | 格式 |
|---------|---------|------|
| 评测数据 | B站评测视频字幕（.srt/.txt） | 直接丢文件 |
| 评论区 | 微博/小红书/抖音评论区导出 | .csv/.json/.txt |
| 规格参数 | 品牌官网/京东产品页 | .md/.csv |
| 竞品信息 | 手动整理 | .md/.txt |

**最少可用数据**：每个品类2-3个B站评测字幕 + 1个评论区导出，就能开始工作。

## 置信度说明

每次输出末尾会附置信度标注：

```
📋 自检: 5个数值✓ 3个产品✓ 4个来源✓ | 置信度: 高
```

- **高**：所有数据都有知识库出处
- **中**：少量推测已标注
- **低**：较多推断，输出中会提示你
