# 质量自检工具

## 去AI化替换规则

### 禁用句式 → 替换方案

| 禁用 | 替换 | 示例 |
|------|------|------|
| "不是A，而是B" | "A有问题，B才对" / "A不够，B才是关键" | ❌"不是价格问题，而是体验问题" → ✅"价格不是重点，体验才是" |
| "首先/其次/最后" | 直接分点，不加连接词 | ❌"首先性能强，其次续航长" → ✅"性能强。续航长。" |
| "我们可以发现" | "数据显示" / 直接给结论 | ❌"我们可以发现续航提升了30%" → ✅"续航提升30%" |
| "值得注意的是" | 删除，直接说重点 | ❌"值得注意的是温控表现" → ✅"温控是关键" |
| "让我们来看" | 直接展示 | ❌"让我们来看数据" → ✅"数据如下" |
| "不可否认" | 删除 / "确实" | ❌"不可否认影像很强" → ✅"影像确实强" |
| "毋庸置疑" | "确定" / 删除 | ❌"毋庸置疑这是旗舰" → ✅"这就是旗舰" |
| "通过XX我们能够" | "XX说明" / "XX带来" | ❌"通过对比我们能够看出" → ✅"对比说明" |
| "综上所述" | "所以" / 删除 | ❌"综上所述建议入手" → ✅"建议入手" |
| "在XX的背景下" | "XX背景下" / 删除 | ❌"在当前市场竞争的背景下" → ✅"市场竞争激烈" |

### 企业通稿腔检测

如果输出中出现过多空洞的"战略动词"堆砌，标记为"企业通稿腔"，需要重写。

替换原则：
- 空洞动词 → 具体动作（"XX用户"→"让用户能做YY"）
- 抽象概念 → 具体说明（"XX体验"→"体验好在YY"）
- 战略套话 → 说人话（"XX生态"→"把YY和ZZ连起来"）

### 创意文案特别规则

创意文案允许更口语化、更夸张（毕竟是广告），但：
- 仍然不能出现上述AI句式
- 必须有记忆点（看完能记住一句话）
- 必须有传播性（看完想转发/评论）
- 不能是企业通稿和创意文案的混合体

## 事实核查清单

输出涉及技术参数时，逐项检查：

- [ ] 性能数据：是否引用了实测跑分/帧率？来源标注了吗？
- [ ] 续航数据：是官方数据还是实测？标注了吗？
- [ ] 影像/音质描述：是主观评价还是可量化指标？
- [ ] 价格信息：是否是最新的官方价格？
- [ ] 竞品对比数据：两个产品的数据来源是否一致？
- [ ] 功能描述：是否覆盖了"什么情况下不能用"的限制条件？

## 自检报告格式

每次输出末尾附上：

```
📋 自检: {N}个数值✓ {N}个产品✓ {N}个来源✓ | 置信度:{高/中/低}
```

- **数值✓**: 输出中的所有数字都已核实
- **产品✓**: 涉及的产品信息来自知识库
- **来源✓**: 每个数据点都标注了出处
- **置信度**: 高(全有出处) / 中(少量推测已标注) / 低(较多推断)

置信度为"低"时，必须在输出中提示用户。
