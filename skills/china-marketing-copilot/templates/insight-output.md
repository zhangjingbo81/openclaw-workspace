# 洞察输出模板

适用于竞品分析、数据洞察、横评对比等任务。

```
{分析主题} — 洞察报告

一句话结论
{最核心的发现}

关键发现
{发现1} — 数据来源: [{评测KOL名}|{平台}|{评测标题}]
{发现2} ...
{发现3} ...

数据支撑
{引用知识库中的具体数据和原文}

受众视角
发烧友怎么看: {观点}
普通用户怎么看: {观点}
价格敏感用户怎么看: {观点}

可行动建议
1. {具体建议，可执行}
2. {具体建议}
3. {具体建议}

置信度
{高/中/低} — {原因说明}
```

## 横评对比专用格式

当任务涉及多产品对比时，额外输出：

```
对比矩阵

| 维度 | {产品A} | {产品B} | {产品C} | 胜出 |
|------|---------|---------|---------|------|
| 性能 | {数据} | {数据} | {数据} | {谁} |
| 续航 | {数据} | {数据} | {数据} | {谁} |
| 影像/音质 | {数据} | {数据} | {数据} | {谁} |
| 设计 | {主观} | {主观} | {主观} | {谁} |
| 性价比 | {判断} | {判断} | {判断} | {谁} |

推荐场景
- 追求{XX}的用户 → {产品}
- 预算{XX}的用户 → {产品}
- {特定需求}的用户 → {产品}
```

### 横评时的注意事项

- 对比数据必须同源（同一评测机构的数据，不同机构的数据需标注差异）
- 主观维度（设计/手感）标注为"主观"
- 胜出判断要有依据，不能凭感觉
- 每个维度如涉及风险传播话术，对照 [industry-memes.md](../ecosystem/industry-memes.md) 检查

## 数据来源标注规范

每个数据点必须标注来源类型：

| 来源类型 | 标注方式 | 可信度 |
|---------|---------|--------|
| 知识库文件 | [{文件名}] | 高 |
| 行业报告 | [{报告名}+{机构}] | 高 |
| KOL评测 | [{KOL名}+{平台}+{评测标题}] | 中 |
| 评论区采集 | [{平台}+{量级}] | 中 |
| 公开新闻 | [{媒体名}] | 中 |
| 推测/推断 | [推测] | 低 |
