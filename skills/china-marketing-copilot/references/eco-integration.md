# 生态集成指南

本 Skill 可与以下外部能力配合，增强实时性和数据覆盖。

---

## web-search / felo-search

**用途**: 实时搜索竞品信息、行业热点、最新价格

**典型场景**:
| 场景 | 搜索关键词示例 |
|------|--------------|
| 竞品动态 | "{竞品品牌} {品类} 发布 新品" |
| 行业热点 | "{品类} 热搜 争议 翻车" |
| 最新价格 | "{产品名} 京东 价格" |
| 评测解禁 | "{产品名} 评测 解禁" |

**注意事项**:
- 搜索结果需要经过数据纪律审核——网页数据不等于知识库数据
- 价格信息时效性强，使用时标注"[实时搜索，{日期}]"
- 搜索结果不直接写入知识库，需经 DataProcessor 清洗后才能入库

---

## browser-use

**用途**: 爬取平台热搜/话题/评论区数据

**典型场景**:
| 场景 | 目标URL |
|------|---------|
| 微博热搜 | weibo.com/hot/search |
| 小红书种草 | 搜索 "{品类} {产品} 推荐" |
| 抖音话题 | 搜索 "#{话题标签}" |
| B站弹幕/评论 | 视频页评论区 |

**注意事项**:
- 爬取的数据同样需要经 DataProcessor 清洗
- 平台反爬策略不同，成功率不稳定
- 爬取内容不自动进入知识库

---

## summarize

**用途**: 处理长评测视频字幕、长篇行业报告

**典型场景**:
- 1小时+的评测视频字幕 → 提取关键发现
- 行业报告全文 → 提取核心数据点
- 多条评论区数据 → 提取高频观点

**与 DataProcessor 的关系**:
- summarize 是"压缩"——长文本→短摘要
- DataProcessor 是"结构化"——原始文本→知识库格式
- 可以串联使用：先 summarize 压缩，再 DataProcessor 结构化
